托管到GitHub Pages

1. 新建Repo

在GitHub上新建一个Repo，并且名称命名为username.github.io

2. 将本地代码库关联并Push到远程Repo

GitHub Actions实现自动部署

每次修改完博客后都需要重新生成静态文件并部署，我们可以利用GitHub Actions实现自动化，只要将代码push到远端就能自动部署，真正实现”所写即所得”。

我们需要在.github/workflows路径下新建deploy.yml文件，并Push到Repo中

每次master提交后，就会自动触发流水线，生成静态文件并部署到博客Repo

​

自定义博客域名

GitHub Pages默认的博客域名是username.github.io,我们可以为其绑定自定义的域名，本人采用阿里云。

1. 购买域名

在阿里云域名中心购买域名（需要填写个人信息并实名认证,审核过后即可使用）

2. 解析设置

在控制台打开自己想要配置的域名，进行解析配置

添加两条解析配置到github博客域名

3. CNAME文件

在source文件夹中新建CNAME文件（在source中新建CNAME可以防止每次部署文件被覆盖），填写自定义博客域名并提交，待生效后即可从自定义域名中访问博客

添加Https证书

默认的自定义域名是不带Https的，为了能够在浏览器地址栏里看到小锁头，我们还要做接下来的配置

1. 利用阿里云SSL证书

这里我们可以利用阿里云数字证书管理服务购买免费的SSL证书，购买完毕后点击创建证书会生成一个新的证书，之后点击证书申请，为我们新建的证书做一些配置。

将证书绑定到自己的域名下，并填写相关信息，审核生效后即可。

到这一步我们已经可以通过Https协议正常访问我们的自定义博客站点了。

2. 开启强制Https访问

如果我们想强制走Https访问，需要在GitHub里做些配置，只需要勾选设置中Enforce HTTPS 这个选项即可

如果Https不生效，我们可以先把自定义域名去除，生效后重新加回来即可

哈希函数

发表于 2022-02-23 更新于 2022-03-02

基本概念

加密散列函数( CHF ) 是一种数学算法，可将任意大小的数据（通常称为“消息”）映射到固定大小的位数组（“散列值”、“散列”或“消息摘要” ）。它是一种单向函数，即实际上不可能对其计算求逆或逆向的函数。
加密散列函数必须是确定性的，这意味着相同的消息总是产生相同的散列。理想情况下，它还应该具有以下属性：

  • 可以快速计算任何给定消息的哈希值
  • 生成产生给定哈希值的消息是不可行的（即反转生成给定哈希值的过程）
  • 找到具有相同哈希值的两条不同消息是不可行的
  • 对消息的微小更改应该会如此广泛地更改哈希值，以至于新的哈希值看起来与旧的哈希值不相关

HASH算法分类和对应的位长

MD5 SHA1 SHA224 SHA256 SHA384 SHA512
128 160 224 256 384 512

应用

  • 数字签名、消息认证码(MAC)等认证中
  • 在hash table中索引数据
  • 检测重复数据或唯一标识文件
  • 作为校验和检测意外数据损坏
